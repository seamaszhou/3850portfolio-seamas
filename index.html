What is Reverse Jenga

Reverse Jenga is a special version of traditional Jenga Game. The purpose of Reverse Jenga is helping young generation studying stress control during the funny Jenga Game. By using IOT technology particle photon board and microcontroller Arduino with several pulse sensors, Reverse Jenga can determine user’s stress level and using the wireless network to control special Jenga block to visualization player’s emotion by vibrating blocks.

How the game play

The basic rule of Reverse Jenga is same as classic Jenga. The Jenga tower consists 12 blocks(There was supposed to be more wood, but we only made so much for cost reasons)separated in 4 layers, users need removing one each time and stacking them to a higher level. Players only allowed use one hand to remove the block because they need to use another hand connected with the pulse sensor for detecting their heartbeat.

The Computer will handle and analysis user’s heartbeat change and determining user is nervous or not. If the machine thinks the user is under stressed it will send a signal to those block installed with vibrating module over TCP (Transmission Control Protocol) network by Node JavaScript. Those blocks will start vibrating until user overcome the stress situation, or when their heartbeat get normal.

Final Product

Our final product achieved the MVP（Minimum viable product ）of the idea. We have two pulse sensors for the player. The micron control board（Arduino） can distinguish which player’s heartbeat is increasingly influenced by the stress also reflect two led light will increase fade in speed depends on user’s heartbeat frequency. The vibration module is strong enough to influence the game and increase the difficulty. The Players will have to control their emotions under pressure to complete the game.

In Exhibition

In the exhibition, our products attract a large number of spectators to play. People say our project’s goal is simple, clear and significant. By using Jenga, this traditional game allows most people to get started quickly and enjoy the game. Also, people like the way we visualize stress through block vibrations, which makes the game more difficult, interest, and lets players learn that to be successful, they have to learn to overcome nervousness which is our purpose of the project. And even though our original target was children, many adult users at the show also found our Jenga very helpful in learning to relieve stress, and quite a few people came to visit our products many times.

1.Idea Generation

In the first studio after we arranged to a group, We start brainstorm immediately. The topic of our team assigned is changing behaviours, We list the behaviours that people need to change such as procrastination, stay up late, irritability and so on. At first, we worked around irritability control and emotion visualization. We envision creating a block to reflect emotions such as anger and frustration. This block will be used in common table games where a user reacts negatively to the performance of another player (usually due to some mistake) and the box will display it. This will make the offending teammate realize that he or she is annoying others and warn the user that his or her emotions are affecting the game so he or she needs to control their emotion to keep the game going on

After did research and got the response from teaching team and students, we realized the purpose for emotion control still not clear enough, also determine user’s emotions is difficult to reach. So we changed the direction of the idea from emotion visualization to stress control. We still keep the idea of visualizing the user's emotions, but it's much less difficult because we only need to visualize the stress emotion this time. In order to create a tense simulation scene for the user, we used the traditional game: Jenga. At the same time, because our target users are early-education children, our Jenga game will be educational and interesting

2. First inteartion

The early version of reverse Jenga was building by using the normal Jenga block, vibration motor, pulse sensor and Arduino board. Limited by the Arduino, the first version of Jenga is not wireless. We use several Jump wires to extends and plugged materials into Arduino. The overall product is not beautiful enough and can wire be a great distraction to play.

In the early version, the Arduino board only can connect with one pulse sensor and can’t reflect BPM very well. After doing plenty of user testing with the pulse sensor, we collected huge data and found the most suitable finger user should wear and way to fix it so that it could accurately feedback the real heartbeat.

3. Second iteration

After the first iteration, we decided to build our blocks wireless for making the game more interactive. At first, we are struggling with using Arduino to achieve the wireless function. But thanks to Lorna and the teaching team, they show us a wonderful way to solve the wireless issue: Using particle photon board.

Particle photon is an IOT platform, we found the photon board can transferring and handling data by using TCP network. We use Node Java script to transferring the data between Arduino and Photon. Arduino will detecting user’s heart beat and sending to photon board. During the process of sending, photon board will calculating and determining the user is stressed or not. And the toggle of vibration motor on photon will be turned on by if the user is stressed out.

Also, we successfully implemented multi-pulse sensors and Arduino can determine the Bpm is from which user also send the data to photon separately.

5. Reflection

1. During the process of developing, I was stuck with a really simple bug of data transfer between photon and Arduino for few days. The reason I can’t make this function work is because of when I programming that part I followed a tutorial online and didn’t totally understand how the actual function work. This warning me during the process of programming, we should depth understand the code and language instead of copy and paste tutorial code online. After I realized how easy the bug can be solved I felt real shame and re-developed most function by using my knowledge.

2. Even the final product works well during the exhibition, but I am not really happy with the pulse sensor. We use the open source pulse sensor library and kit from the internet, although most of the time is working good, the data will occasionally be inaccurate sometimes. We shall consider and look for more solution on the pulse sensor.

3. In the final product, we did not use a good variety of ways to show the user's nervousness (except wood block vibration). Many users also feedback that they do not know whose nervousness affected the game. We originally had two LED lights to show the user's heartbeat changes, but for the overall beauty of the product removed this feature. We should consider visualizing the stress emotion by using led tail.

4. Another thing we don't consider is that there are a lot of users coming to visit our products on the day of the exhibition. So the Jenga block will fall many times, causing a few blocks to broken and take a while to repair. Also, some batteries do not have enough power, so unfortunately in the middle of the exhibition need to get some batteries out from the block to charge.
